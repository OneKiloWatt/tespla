# PAGES / PLAN_WIZARD

計画作成ウィザード（`/plan/new`）の画面仕様。5 ステップで構成される。

## ステップ構成

```
Step 1: テスト情報
Step 2: 自動 / 手動の選択
Step 3: 自動設定          ← 手動モード選択時はスキップ
Step 4: 勉強配分の確認・調整
Step 5: 確認・保存
```

## Step 1: テスト情報

入力項目と初期値：

| 項目 | 初期値 |
|---|---|
| テスト名 | 作成タイミングから 1〜3 学期の中間 / 期末を自動で設定 |
| テスト開始日 | 作成日から 1 か月後 |
| テスト終了日 | 作成日から 1 か月後 |
| テスト教科 | なし（複数選択） |
| テスト日ごとの教科割り当て | なし |

教科の選択肢：中学校で習う標準教科をデフォルトで用意する。ない場合はユーザーが選択肢を追加できる。

テスト日ごとの教科割り当て：選んだテスト教科から、日付ごとに教科を選択する。

## Step 2: 自動 / 手動の選択

- 「自動で作成する」を選んだ場合：Step 3 へ進む
- 「手動で作成する」を選んだ場合：Step 3 をスキップして Step 4 へ進む

## Step 3: 自動設定（自動モードのみ）

入力項目と初期値：

| 項目 | 初期値 |
|---|---|
| 平日の勉強時間 | 90 分 |
| 平日の勉強時間（部活あり） | 60 分 |
| 休日の勉強時間 | 120 分 |
| 部活がある曜日 | 月〜金 |
| テスト 1 週間前は部活なし | ON |

詳細なアルゴリズム仕様は `doc/AUTO_PLAN_LOGIC.md` を参照。

## Step 4: 勉強配分の確認・調整

- 自動生成された配分と手動で入力した配分を区別せずに表示する
- 内容を自由に修正できる

## Step 5: 確認・保存

- 入力内容を確認して保存する
- 保存後、計画はメインストアに反映される
